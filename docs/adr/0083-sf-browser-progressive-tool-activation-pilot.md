# ADR 0083: SF Browser Progressive Tool Activation Pilot

Status: accepted

SF Pi will evaluate Pi's dynamic tool loading through one bounded SF Browser pilot after Active-Branch Context Projection is correct. The initial active Browser set keeps org opening and route resolution visible; a successful workflow entry call additively activates snapshot, interaction, wait, editor, and evidence tools for the next model request. Tools remain registered, and activation changes visibility only—not Guardrail mediation, user confirmation, headless restrictions, or Pi's authoritative tool exclusions.

The pilot must prove next-turn availability, unsupported-model fallback, exclusion preservation, Guardrail interception, resume/compaction reconstruction, and measurable tool-schema or cache benefit. SF Pi will not set provider-specific deferred-tool compatibility flags without live route proof and will not introduce a package-wide capability router unless the pilot demonstrates enough leverage to justify its additional routing and eligibility interface.
