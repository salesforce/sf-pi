# ADR 0083: SF Browser Progressive Tool Activation Pilot

Status: stopped before implementation; eager tool activation retained

SF Pi will evaluate Pi's dynamic tool loading through one bounded SF Browser pilot after Active-Branch Context Projection is correct. The initial active Browser set keeps org opening and route resolution visible; a successful workflow entry call additively activates snapshot, interaction, wait, editor, and evidence tools for the next model request. Tools remain registered, and activation changes visibility only—not Guardrail mediation, user confirmation, headless restrictions, or Pi's authoritative tool exclusions.

The proposed pilot required next-turn availability, unsupported-model fallback, exclusion preservation, Guardrail interception, resume/compaction reconstruction, and measurable tool-schema or cache benefit. Before implementation, the product decision was to retain Pi's normal eager tool activation and avoid dynamic tool loading in the current adoption program. No Browser production code, compatibility flag, loader, capability router, or state migration landed. Dynamic activation can be reconsidered only as a separately authorized future feature.
