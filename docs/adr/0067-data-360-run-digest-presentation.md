# Data 360 tools use Run Digests for presentation

`data360_*` tools will present results through a compact **Data 360 Run Digest** plus human-facing **Data 360 Result Card** instead of using generic raw JSON summaries as the primary output. The first seam is the v2 `buildToolResult()` presenter so execution behavior stays stable while every family tool can gain better traceability; raw evidence is preserved as **Data 360 Artifacts** for org-touching runs, SQL/runbooks, journeys, OTel exports, dry-run safety resolutions, and large responses. This trades raw inline payloads for Agent Script-style context efficiency and human UX: `output_mode` controls preview density, not whether the model receives a semantic digest, and rich adapters can be added incrementally starting with Observe.
