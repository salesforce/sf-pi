# ADR 0024: SF Code Analyzer onboarding is cache-first and dismissible

SF Code Analyzer is default-enabled with readiness-gated automatic behavior. It should recommend installation and setup similarly to SF LSP, but must not add work to the startup critical path. The extension records cached readiness for the Code Analyzer CLI plugin and engine prerequisites, surfaces that cache in SF Welcome and `/sf-code-analyzer`, contributes full detail to `/sf-pi doctor`, and refreshes it only through deferred background work or explicit doctor/setup actions. Users can dismiss install recommendations and that dismissal is remembered until a meaningful version or readiness change re-arms the nudge. V1 does not add a permanent SF DevBar segment because Code Analyzer is a scan workflow, not a continuously running language service. If Code Analyzer is not ready, no automatic local or ApexGuru scans run; users see dismissible setup recommendations instead.
