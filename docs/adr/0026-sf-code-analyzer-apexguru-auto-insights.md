# ADR 0026: ApexGuru auto insights are default-on when cached availability is enabled

SF Code Analyzer includes ApexGuru in v1 as an explicit action and as an opportunistic automatic stage that is enabled by default when cached readiness for the current org/user says ApexGuru is `enabled`. ApexGuru never runs on startup, never performs live readiness checks on the boot path, and never runs automatically when readiness is unknown, unavailable, stale beyond the automatic-scan threshold, keyed to a different org id / user id / instance URL / API version tuple, or disabled by a global or project preference. When eligible, the automatic stage runs after the local deferred Code Analyzer scan and scans changed production Apex files sequentially in deterministic path order rather than selecting one file through heuristics; Apex test classes are scanned only through explicit ApexGuru runs. Automatic ApexGuru has a 60-second total batch budget per deferred quality pass, and any unscanned files are reported with an explicit command to continue. It may run even when the local scan found high-severity findings because additional performance guidance can help the LLM repair code more effectively. Because ApexGuru sends Apex class content to an org-backed service, SF Code Analyzer makes this visible in SF Welcome, the `/sf-code-analyzer` panel, and the first automatic ApexGuru transcript row for a project, but does not show a blocking per-run confirmation dialog.
