# ADR 0077: GPT-5.6 Sol as Gateway Default

SF Pi will make `gpt-5.6-sol` the SF LLM Gateway default model for new or obsolete gateway defaults because gateway discovery and live probes show the non-Bedrock GPT-5.6 Sol route supports 1M-class context, 128K output, priority traffic, and Pi `max` thinking mapped to wire `max`. Static gateway presets will include all discovered GPT-5.6 variants, while the gateway fallback model should move to `claude-sonnet-5`. Bedrock GPT-5.6 variants stay non-default and must suppress priority service-tier payloads because the gateway rejects `service_tier: "priority"` on those routes. The former Opus defaults remain selectable models; they are no longer product defaults. Existing gateway defaults that still point at old product defaults or their aliases, including Opus 4.8 and Opus 4.7 variants, should migrate to `gpt-5.6-sol` exactly once per settings file; after that one-time migration, users can choose those Opus models again without SF Pi rewriting them on startup. Existing users who have explicitly chosen a different non-default model should keep that choice unless they run an explicit set-default flow. Model defaulting does not imply thinking-level defaulting: `max` remains an advertised model capability, while Pi settings and the user own the active thinking level; Gateway lifecycle hooks must not select it implicitly.
